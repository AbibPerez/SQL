# 1.3 - Tipos de Datos SQL
En SQL, a cada columna se le debe asignar un tipo de dato que defina el tipo de datos que puede almacenar, como enteros, fechas, texto o valores binarios. Elegir el tipo de dato correcto es crucial para la integridad de los datos, el rendimiento de las consultas y la indexación eficiente.

---
## 1. Tipos de datos numéricos
Los tipos de datos numéricos son fundamentales para el diseño de bases de datos y se utilizan para almacenar números, ya sean enteros, decimales o de punto flotante. Estos tipos de datos permiten operaciones matemáticas como suma, resta, multiplicación y división, lo que los hace esenciales para gestionar datos financieros, científicos y analíticos.

### Tipo de dato numérico exacto
Se utilizan tipos numéricos exactos cuando se necesitan valores numéricos precisos, como para datos financieros, cantidades y conteos. Los siguientes son algunos comunes.

| Tipo de Datos | Descripción                           |
| ------------- | ------------------------------------- |
| `BIGINT`      | Valores enteros grandes               |
| `INT`         | Valores enteros estándar              |
| `SMALLINT`    | Valores enteros pequeños              |
| `TINYINT`     | Valores enteros muy pequeños          |
| `DECIMAL`     | Valores exactos                       |
| `NUMERIC`     | Valores precisos, similar a `DECIMAL` |

##### Ejemplo
```SQL
    CREATE TABLE ProductosOferta (
        ProductoID INT PRIMARY KEY,
        Cantidad SMALLINT,
        PrecioUnitario DECIMAL(10,2),
        MontoTotal DECIMAL(10,2)
    );
```

### Tipo de dato numérico exacto
Estos tipos se utilizan para almacenar valores aproximados, como mediciones científicas o grandes rangos de datos que no requieren precisión exacta.

| Tipo de Datos | Descripción                                |
| ------------- | ------------------------------------------ |
| `FLOAT`       | Valores numéricos aproximados              |
| `REAL`        | Similar a `FLOAT` pero con menos precisión |

##### Ejemplo
```SQL
    CREATE TABLE Mediciones (
        SensorID INT,
        Temperatura FLOAT,
        Humedad REAL
    );
```

## 2. Tipos de datos de caracteres y cadenas
Los tipos de datos de caracteres se utilizan para almacenar texto o datos basados en caracteres. La elección entre tipos de datos de longitud fija y variable depende de la naturaleza de tus datos.

| Tipo de Datos  | Descripción                                             |
| -------------- | ------------------------------------------------------- |
| `Char`         | Caracter de longitud fija máxima de 8000 caracteres     |
| `Varchar`      | Caracter de longitud variable máxima de 8000 caracteres |
| `Varchar(max)` | La longitud máxima de `2^31 - 1` caracteres             |
| `Text`         | La longitud máxima es de 2,127,483,647 caracteres       |

##### Ejemplo
```SQL
    CREATE TABLE InfoEmpleado (
        EmpleadoID INT PRIMARY KEY,
        PrimerNombre VARCHAR(50), 
        Apellido CHAR(30),
        BIO TEXT
    );
```

### Tipos de cadenas de caracteres Unicode
Los tipos de datos Unicode se utilizan para almacenar caracteres de cualquier idioma, soportando una mayor variedad de caracteres. Estos se muestran en la siguiente tabla.

| Tipo de Datos    | Descripción                                                                                     |
| ---------------- | ----------------------------------------------------------------------------------------------- |
| `Nchar`          | Caracteres Unicode con longitud fija máxima de 4000 caracteres                                  |
| `Nvarchar`       | Caracteres Unicode de longitud variable máxima de 4000 caracteres                               |
| `Nvarchar (max)` | Caracteres Unicode de longitud variable máxima de 2^31 - 1 caracteres (solo en SQL Server 2005) |

##### Ejemplo
```SQL
    CREATE TABLE UsuariosInternacionales (
        UsuarioID INT PRIMARY KEY,
        NombreCompleto NVARCHAR (100),
        Pais NCHAR (50)
    );
```

## 3. Tipo de datos de Fecha y Hora
SQL proporciona varios tipos de datos para almacenar información de fecha y hora. Son esenciales para gestionar marcas de tiempo, eventos y consultas basadas en tiempos.

| Tipo de Datos | Descripción                                              | Tamaño de almacenamiento |
| ------------- | -------------------------------------------------------- | ------------------------ |
| `DATE`        | Almacena los datos de fecha (año, mes, día)              | 3 bytes                  |
| `TIME`        | Almacena los datos del tiempo (horas, minutos, segundos) | 3 bytes                  |
| `DATETIME`    | Almacena los datos de fecha y hora                       | 8 bytes                  |

##### Ejemplo 
```SQL
    CREATE TABLE Pedidos (
        PedidoID INT PRIMARY KEY,
        PedidoFecha DATE,
        PedidoHora TIME,
        EnviadoA DATETIME
    );
```

## 4. Tipo de datos binarios
Los tipos de datos binarios se utilizan para almacenar datos binarios como imagenes, videos u otros tipos de archivos.

| Tipo de Datos | Descripción                           | Longitud máxima     |
| ------------- | ------------------------------------- | ------------------- |
| `Binary`      | Datos binarios de longitud fija       | 8000 bytes          |
| `VarBinary`   | Datos binarios de longitud variable   | 8000 bytes          |
| `Image`       | Almacena datos binarios como imagenes | 2,147,483,647 bytes |

##### Ejemplo
```SQL
    CREATE TABLE ImgProductos (
        ImagenID PRIMARY KEY,
        ImagenNombre VARCHAR (100),
        ImagenDatos VARBINARY(MAX)
    );
```

## 5. Tipo de dato booleano
Los tipos de datos booleanos se utilizan para almacenar valores lógicos, normalmente verdadero o falso. Se utiliza comúnmente para campos bandera o condiciones binarias. En SQLite, no hay `BOOLEANBIT` o algún tipo de dato separado. En su lugar, los valores booleanos se almacenan usando `INTEGER`, donde: 
+ `1` representa `TRUE`
+ `0` representa `FALSE`

##### Ejemplo
```SQL
    CREATE TABLE EstadoUsuario (
        UsuarioID INT PRIMARY KEY
        Activo INTEGER,
        Verificado INTEGER
    );
```

## 6. Tipos de datos especiales
SQL también soporta algunos tipos de datos especializados para casos de uso avanzados:
+ __Tipo de dato XML:__ Se utiliza para almacenar datos XML y manipular estructuras XML en la base de datos. 
##### Ejemplo
```SQL
    CREATE TABLE RegistrosXML (
        RegistroID INT PRIMARY KEY,
        ConfigData XML
    );
```
+ __Tipos de datos espaciales (geometría):__ Almacena datos espaciales planos, como puntos, líneas y polígonos, en una tabla de base de datos.

##### Ejemplo
```SQL
    CREATE TABLE Ubicaciones (
        UbicacionID INT PRIMARY KEY,
        Area GEOMETRY
    )
```
