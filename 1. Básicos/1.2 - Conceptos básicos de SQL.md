# 1.2 - Conceptos básicos de SQL
Aprende los conceptos fundamentales de SQL, esenciales para cualquiera que trabaje con bases de datos relacionales. Esta sección cubre la sintaxis, los comandos y los elementos clave para empezar a consultar y gestionar datos de forma eficaz.

## Introducción
El Lenguaje de Consulta Estructurada (SQL) es el lenguaje estándar utilizado para interactuar con bases de datos relacionales.

+ Permite a los usuarios almacenar, recuperar, actualizar y gestionar datos de forma eficiente mediante comandos sencillos.
+ Es conocido por su sintaxis fácil de usar y sus potentes capacidades; SQL se utiliza ampliamente en diversos sectores.

## Funcionamiento de SQL
Interactuamos con bases de datos usando consultas SQL. Herramientas de SGBD como MySQL y SQL Server tienen su propio motor SQL y una interfaz donde los usuarios pueden escribir y ejecutar consultas SQL.

A continuación, se presentan los pasos detallados implicados en la ejecución de la consulta SQL.
+ __Entrada.__ El usuario envía una consulta SQL (como `SELECT` o `INSERT`).
+ __Análisis sintáctico.__ El sistema comprueba si la consulta está correctamente escrita.
+ __Optimización.__ Elige la forma más rápida de ejecutar la consulta.
+ __Ejecución.__ La base de datos ejecuta la consulta.
+ __Salida.__ El resultado o confirmación se envía de vuelta al usuario.

## Componentes de un sistema SQL
+ __Base de datos:__ Una colección estructurada de datos almacenados en tablas con filas y columnas.
+ __Tablas:__ Almacena datos y aplica reglas para mantenerlos precisos y consistentes.
+ __Índices:__ Ayuda a la base de datos a encontrar datos más rápido sin escanear toda la tabla.
+ __Vistas:__ Tablas virtuales creadas a partir de consultas `SELECT` para facilitar el acceso a los datos.
+ __Procedimientos almacenados:__ Código SQL pre-guardados que ejecuta tareas y mejora el rendimiento y la seguridad.
+ __Transacciones:__ Grupo de acciones SQL que o bien todas tienen éxito o todas fallan en mantener los datos seguros.
+ __Seguridad y permisos:__ Controla quién puede ver, cambiar o gestionar los datos de la base de datos.
+ __Uniones:__ Combina datos de diferentes tablas basándose en relaciones.

## Reglas para escribir consultas SQL
Existen ciertas reglas para SQL que garantizarían la consistencia y funcionalidad entre bases de datos. Siguiendo estas reglas, las consultas estarán bien formadas y ejecutadas en cualquier base de datos.
+ __Punto y coma (`:`):__ Termina una sentencia SQL.
+ __Insensible a mayúsculas y minúsculas:__ Palabras clave SQL como `INSERT` y `SELECT` no son sensibles a mayúsculas.
+ __Espacio en blanco:__ Se permiten espacios y líneas nuevas para mejorar la legibilidad.
+ __Palabras reservadas:__ No uses palabras clave SQL como nombres, ni las escribas entre comillas o backticks.

### Comentarios
+  __Línea única:__ `-- comentario`
+  __Multilínea:__ `/* comentario */`

### Restricciones de datos
Utiliza `NOT NULL`, `UNIQUE`, `PRIMARY KEY`, etc. para garantizar la precisión de los datos.

### Valores de cadena
Encierra cadenas entre comillas ('cadena')

### Reglas de nombre
+ Empieza con una letra
+ Máximo de 30 caracteres
+ Solo se permite el uso de letras, números y guiones bajos (`_`)

## Consultas SQL
Los comandos de Lenguaje de Consulta Estructurada (SQL) son instrucciones estandarizadas que utilizan los desarrolladores para interactuar con los datos almacenados en __bases de datos relacionales__. Estos comandos permiten la creación, manipulación, recuperación y control de datos, así como de estructuras de bases de datos. Los comandos SQL se categorizan según sus funcionalidades específicas:<br>

---
### 1. Lenguaje de Definición de Datos
Los comandos DDL (Lenguaje de Definición de Datos) se utilizan para crear, modificar y eliminar objetos de base de datos. Los ingenieros de bases de datos utilizan comandos como `CREATE` para crear tablas, vistas e índices, y `ALTER` o `DROP` para modificarlos o eliminarlos.

| Comando    | Descripción                                                                              |
| ---------- | ---------------------------------------------------------------------------------------- |
| `CREATE`   | Crea una nueva tabla, una vista en una tabla u otro objeto en la base de datos           |
| `ALTER`    | Modifica un objeto de base de datos existente, como una tabla                            |
| `DROP`     | Elimina una tabla entera, una vista de tabla u otros objetos de la base de datos         |
| `TRUNCATE` | Elimina todos los registros de una tabla pero mantiene la estructura de la tabla intacta |
| `RENAME`   | Cambia el nombre de un objeto de base de datos existente, como una tabla                 |

---
### 2. Lenguaje de Manipulación de Datos
Una base de datos relacional puede actualizarse con nuevos datos utilizando sentencias del lenguaje de manipulación de datos (DML). EL comando `INSERT`, por ejemplo, es utilizado por una aplicación para añadir un nuevo registro a la base de datos. 

| Comando  | Descripción            |
| -------- | ---------------------- |
| `INSERT` | Crea un registro       |
| `UPDATE` | Modifica los registros |
| `DELETE` | Borra registros        |

---
### 3. Lenguaje de Consulta de Datos
Las instrucciones de recuperación de datos se escriben en el lenguaje de consulta de datos (DQL), que se utiliza para acceder a bases de datos relacionales. El comando `SELECT` es utilizado por programas de software para filtrar y devolver resultados particulares de una tabla SQL.

---
### 4. Lenguaje de Control de Datos
Los comando DCL gestionan el acceso de los usuarios a la base de datos otorgando o revocando permisos. Los administradores de bases de datos utilizan DCL para hacer cumplir la seguridad y controlar el acceso a objetos de bases de datos.

| Comando  | Descripción                                  |
| -------- | -------------------------------------------- |
| `GRANT`  | Otorga un privilegio al usuario              |
| `REVOKE` | Revoca los privilegios concedidos al usuario |

---
### 5. Lenguaje de Control de Transacciones
Los comandos TCL gestionan las transacciones en bases de datos relacionales, garantizando la integridad y consistencia de los datos. Estos comandos se utilizan para confirmar cambios o revertir operaciones en caso de errores.

| Comando     | Descripción                                                                                                                                            |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `COMMIT`    | Guarda todos los cambios realizados durante la transacción actual de forma permanente. Algunas bases de datos permiten configurar la auto-confirmación |
| `ROLLBACK ` | Revierte los cambios realizados durante la transacción actual, asegurando que no se guarden cambios no deseados                                        |
| `SAVEPOINT` | Establece un punto dentro de una transacción al que los cambios pueden revertirse, permitiendo reversiones parciales                                   |

---
## Beneficios de SQL
+ __Eficiente:__ Funciona rápido e incluso con datos grandes y complejos
+ __Estándar:__ Funciona en la mayoría de los sistemas de bases de datos
+ __Escalable:__ Puede gestionar bases de datos pequeñas a muy grandes
+ __Flexible:__ Soporta lógica avanzada usando extensiones como __PL/SQL__ o __T-SQL__.

## Limitaciones de SQL
+ __Complejo:__ La optimización y ajustes avanzados pueden ser difíciles.
+ __Problemas de escalabilidad:__ No es ideal para datos grandes o no estructurados.
+ __Diferentes versiones:__ Algunas funciones de SQL varían según la base de datos.
+ __Sin tiempo real:__ SQL tradicional no está diseñado para análisis en tiempo real.

## Aplicaciones de SQL en la vida real
+ __Comercio electrónico:__ Gestionar pedidos de clientes, catálogos de productos e inventario.
+ __Atención sanitaria:__ Conserva los registros de los pacientes y las horas de las citas.
+ __Banca:__ Analiza los historiales de transacciones y genera informes financieros.
+ __Desarrollo web:__ Dinámica de sitios web con contenido específico para usuarios.
+ __Aprendizaje automático y Ciencia de Datos:__ Combina bases de datos SQL con herramientas como Python, R y TensorFlow para agilizar flujos de trabajo de aprendizaje automático.
